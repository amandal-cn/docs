## Creating a Hive from API Specification

Follow these steps to create a new hive using your API specification.

### Step 1: Prepare Your API Spec

Your API specification should be in OpenAPI (Swagger) format. Ensure it includes:
- Endpoint definitions
- Request/response schemas
- Authentication requirements
- API descriptions

### Step 2: Create New Hive

1. Navigate to the "Create Hive" section
2. Select "From API Specification"
3. Enter basic information:
   - Hive name
   - Description
   - Tags (for discoverability)
   - Version

### Step 3: Upload API Specification

<Note>
We support OpenAPI versions 2.0 and 3.0+
</Note>

You can either:
- Upload an OpenAPI file
- Paste your specification directly
- Provide a URL to your specification

### Step 4: Configure Authentication

Select the authentication type required for your API:
- API Key
- OAuth 2.0
- Basic Auth
- Custom

### Step 5: Review and Publish

- Review the generated capabilities
- Test the endpoints
- Publish your hive to the marketplace

## Best Practices

- Provide clear descriptions for all endpoints
- Include example responses
- Use meaningful tags for better organization
- Test all endpoints before publishing
